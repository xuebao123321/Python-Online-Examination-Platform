## 批次十：错题重练功能

```
请为错题本增加「错题重练」功能，让学生能把错题重新组成试卷练习，确保系统完整可运行：

**改动1：错题本增加「🎯 错题重练」入口**
修改 `page_student_wrong()` 页面，在标题下方、错题列表上方增加一个操作栏：

- 显示「🎯 错题重练」按钮（primary 类型，醒目）
- 按钮上方显示错题总数统计，例如：「共 12 道错题，可选择重练」
- 点击后弹出配置面板（或用 st.expander 展开）：
  - 题目来源选择：单选 / 判断 / 全部（默认全部，排除编程题）
  - 题目数量：全部 / 自定义（滑块 5-50 道）
  - 顺序：顺序 / 随机（默认随机）
  - 模式：练习模式 / 模拟考试（默认练习模式）
    - 练习模式：无倒计时，每题即时显示对错
    - 模拟考试：有倒计时，交卷后统一判分
  - 「🚀 开始重练」按钮

**改动2：重练考试流程**
点击「开始重练」后：

a) 从错题本中筛选题目：
   - 从 `get_wrong_questions()` 获取所有错题（不含编程题）
   - 如果选了「单选」只取单选题，选了「判断」只取判断题
   - 限制题目数量（如果错题不够所选数量，用实际数量并提示）
   - 如果选了随机，打乱题目顺序

b) 复用现有考试逻辑：
   - 创建重练的题目列表，存入 session_state
   - 复用 `_show_exam_questions()` 展示题目
   - 练习模式：复用现有练习模式逻辑（is_practice=True）
   - 模拟考试：创建 attempt 记录（可以在 exam_attempts 中标记 is_retry=1 或存入 bank_id=0 表示重练）
   - 模拟考试不创建正式 attempt 的最简方案：直接复用练习模式 + 倒计时，结果不存入数据库
   - 或者：创建一个特殊的 "错题重练" attempt (bank_id 设为 NULL 或 0)，提交后正常判分但标记为练习

最简实现（推荐）：
- 重练始终使用练习模式逻辑（is_practice=True），交卷后即时判分
- 结果页显示得分和逐题解析，但不保存到历史记录
- 这样改动最小，避免污染正式考试数据

**改动3：重练结果页**
重练交卷后的结果展示：
- 显示总分和正确率
- 每道题展示正确答案和解析（因为重练目的是学习）
- 答对的题从错题本中移除？还是保留？
  - 建议：答对的题在本次重练中标记为 ✅，但不自动从错题本移除
  - 错题本只在实际考试订正后才移除（保持现有逻辑）
- 「🔄 再次重练」和「↩️ 返回错题本」两个按钮

**改动4：错题本增加多选功能**
在错题列表中，每道题左边增加一个 checkbox（用 session_state 管理选中状态）。
- 顶部有「全选 / 取消全选」按钮
- 底部固定栏显示「已选 N 题，🎯 重练已选题」
- 如果没选任何题，则重练全部错题

这样学生可以针对性重练某几道错题。

执行完这些提示词后，系统要完整可运行并且不会有任何bug，认真仔细整理。
```
